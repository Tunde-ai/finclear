<task type="auto">
  <name>Task 1: Fix prisma.config.ts to use DIRECT_URL for migrations</name>
  <files>prisma.config.ts</files>
  <action>
Update `prisma.config.ts` to use `DIRECT_URL` instead of `DATABASE_URL` for the datasource URL.

Current (broken for migrations):
```ts
datasource: {
  url: process.env["DATABASE_URL"]!,
},
```

Change to:
```ts
datasource: {
  url: process.env["DIRECT_URL"]!,
},
```

This is critical because `DATABASE_URL` points at the Supabase connection pooler (PgBouncer in transaction mode), which does not support the DDL commands that Prisma migrations execute. `DIRECT_URL` is the non-pooled connection that supports migrations.

Note: The runtime Prisma client in `lib/prisma.ts` correctly uses `DATABASE_URL` (pooled) and must NOT be changed.
  </action>
  <verify>
Run `grep "DIRECT_URL" prisma.config.ts` to confirm the change.
Run `npx tsc --noEmit prisma.config.ts 2>&1 || echo "check manually"` to verify no type errors (may need full build check).
  </verify>
  <done>prisma.config.ts references DIRECT_URL instead of DATABASE_URL. The file compiles without errors.</done>
</task>

  <instructions>
Complete the following configuration steps. Use DEPLOY.md as your reference.

**Step 1 - Stripe Products and Webhook:**
1. Go to Stripe Dashboard (live mode) -> Products -> Add product
2. Create "Starter" - $19/month recurring
3. Create "Professional" - $49/month recurring
4. Create "Business" - $99/month recurring
5. Copy each product's Price ID (starts with `price_`)
6. Go to Developers -> Webhooks -> Add endpoint
7. Endpoint URL: `https://finclear.app/api/webhooks/stripe`
8. Select events: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`
9. Copy the Webhook Signing Secret (starts with `whsec_`)
10. Go to Settings -> Customer portal -> Enable

**Step 2 - Clerk Webhook:**
1. Go to Clerk Dashboard (production instance) -> Webhooks -> Add Endpoint
2. Endpoint URL: `https://finclear.app/api/webhooks/clerk`
3. Subscribe to: `organizationMembership.created`
4. Copy the Signing Secret

**Step 3 - Resend Domain Verification:**
1. Go to Resend Dashboard -> Domains -> Add Domain
2. Enter `finclear.app`
3. Add the SPF and DKIM DNS records shown to your domain's DNS settings
4. Wait for verification (can take up to 72 hours, usually minutes)

**Step 4 - Vercel Environment Variables:**
1. Go to Vercel Dashboard -> FinClear Project -> Settings -> Environment Variables
2. Set ALL 28 variables listed in DEPLOY.md for the Production environment
3. Key vars to double-check:
   - `DIRECT_URL` (non-pooled Supabase connection - needed for migrations)
   - `DATABASE_URL` (pooled Supabase connection - used at runtime)
   - All 3 `STRIPE_*_PRICE_ID` vars with the Price IDs from Step 1
   - `STRIPE_WEBHOOK_SECRET` from Step 1
   - `CLERK_WEBHOOK_SECRET` from Step 2
   - `RESEND_FROM_EMAIL` set to `FinClear <notifications@finclear.app>`
   - `ENCRYPTION_KEY` generated via `openssl rand -hex 32`
   - `NEXT_PUBLIC_APP_URL` set to `https://finclear.app`
  </instructions>
