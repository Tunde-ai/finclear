  <verify>User confirms migrations were applied successfully (output shows migrations applied or "already applied").</verify>
  <done>All Prisma migrations are applied to the production Supabase database.</done>
  <instructions>
Run the Prisma migration against your production database. You need your DIRECT_URL (the non-pooled Supabase connection string you set in Vercel).

**Option A — Provide DIRECT_URL to Claude:**
Paste your DIRECT_URL value and Claude will run the migration for you:
```
DIRECT_URL="postgresql://..." npx prisma migrate deploy
```

**Option B — Run it yourself:**
```bash
DIRECT_URL="your-non-pooled-supabase-url" npx prisma migrate deploy
```

The output should show each migration being applied, or "Already in sync" if previously applied.

**Important:** Use the DIRECT_URL (non-pooled, typically port 5432), NOT DATABASE_URL (pooled via PgBouncer, typically port 6543). Migrations will fail against the pooled connection.
  </instructions>
  <resume-signal>Paste the migration output, or type "migrations applied" to confirm success.</resume-signal>
